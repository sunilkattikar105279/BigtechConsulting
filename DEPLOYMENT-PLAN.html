STEP 03Choose One Method
Deployment Methods
Three ways to deploy — pick the one that fits your workflow

Method A — Vercel CLI Recommended

Best for developers. Deploys in under 60 seconds from your terminal. Also enables preview deployments for every push.

# 1. Install Vercel CLI globally
npm install -g vercel

# 2. Log in with your Vercel account
vercel login
# → Opens browser, sign in with GitHub

# 3. Navigate to your project root
cd /path/to/bigtechconsulting

# 4. First deploy (follow prompts)
vercel
# When asked:
#   Set up and deploy? → Y
#   Which scope? → your account
#   Link to existing project? → N
#   Project name? → bigtechconsulting
#   In which directory is your code? → ./
#   Want to override settings? → N

# 5. Deploy to production
vercel --prod
# → Live at bigtechconsulting.vercel.app in ~15 seconds

Method B — GitHub Integration

Best for continuous deployment. Every push to main branch auto-deploys to production. Every PR gets a preview URL.

1

Push project to GitHub

cd /path/to/bigtechconsulting
git init
git add .
git commit -m "Initial deploy: BigtechConsulting production site"
git remote add origin https://github.com/sunilkattikar105279/bigtechconsulting.git
git branch -M main
git push -u origin main
2

Import project in Vercel dashboard

Go to vercel.com/new → Click "Import Git Repository" → Select sunilkattikar105279/bigtechconsulting from the list.

3

Configure build settings

In the Vercel import screen, set these values:

Framework PresetOther
Root Directory./ (leave blank)
Build Command(leave empty)
Output Directorypublic
Install Command(leave empty)
💡 Vercel auto-detects the outputDirectory: "public" from your vercel.json, so you may not need to set this manually.
4

Click Deploy

Vercel will deploy in ~15 seconds. You'll get a preview URL like bigtechconsulting.vercel.app. Every future push to main auto-deploys.

Method C — Drag and Drop

No Git, no CLI. Fastest possible deploy — drag the public/ folder directly onto vercel.com/new.

1. Go to vercel.com/new → scroll to "Deploy without Git" section
2. Drag your entire public/ folder into the drop zone
3. Done — live in 30 seconds at a *.vercel.app URL
Note: This method doesn't support auto-deploys. Use Method A or B for ongoing development.

STEP 04Required
Custom Domain Setup
Connect bigtechconsulting.com — Vercel handles SSL automatically
1

Add domain in Vercel dashboard

Go to your project in Vercel → Settings → Domains → Add both:

bigtechconsulting.com
www.bigtechconsulting.com
2

Update DNS records at your registrar

Log into your domain registrar (Namecheap, GoDaddy, etc.) and set these DNS records:

TYPEHOSTVALUE
A@76.76.21.21
CNAMEwwwcname.vercel-dns.com
⏱️ DNS propagation takes 5–60 minutes. Vercel will show a green checkmark when verified.
3

SSL certificate — automatic

Vercel provisions a free Let's Encrypt SSL certificate automatically once DNS propagates. No action required — HTTPS is live within minutes of domain verification.

✅ Your site will be live at https://bigtechconsulting.com with full HTTPS enforced.

💡 Already have a domain elsewhere? If bigtechconsulting.com is registered at GoDaddy/Namecheap with another site on it, just update the A and CNAME records — the old site is replaced. No need to transfer the domain to Vercel unless you want to.

www vs non-www redirect

In your Vercel domain settings, set bigtechconsulting.com as the primary domain and redirect www.bigtechconsulting.com to it (or vice versa). This prevents duplicate content in Google.

STEP 07Already Configured
Performance & CDN
What's already optimized — and what to do next

✅ CDN Edge Caching

vercel.json configures Cache-Control: immutable for all assets. Your CSS and fonts are cached at Vercel's 100+ edge locations globally.

✅ Security Headers

XSS protection, content type sniffing prevention, clickjacking protection, and referrer policy — all configured in vercel.json headers.

✅ Clean URLs

cleanUrls: true in vercel.json means /services/responsible-ai works without the .html extension. Better UX and SEO.

✅ HTTPS Enforced

Vercel auto-provisions SSL and enforces HTTPS. HTTP requests are redirected to HTTPS automatically.

🔲 Google Fonts Preconnect

Already in your HTML heads. Confirm <link rel="preconnect"> tags are present on all pages for faster font loading.

🔲 Image Optimization

When you add images, use WebP format and add loading="lazy" on below-fold images. Vercel Pro includes automatic image optimization.

Preview Deployments

Every non-main branch push gets a unique preview URL (e.g. bigtechconsulting-git-feature-x.vercel.app). Use this to review changes before merging to main.

# Create a preview branch
git checkout -b new-service-page
# make changes...
git push origin new-service-page
# → Vercel creates a preview URL automatically — share for review
git checkout main && git merge new-service-page && git push
# → Promotes to production
